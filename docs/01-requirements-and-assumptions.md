## 비기능 요구사항 해석

- **처리량**: 정량 목표는 주어지지 않음. "DB에는 재고 수량 규모의 쓰기만 도달해야 한다"를 설계 기준으로 삼는다.
- **응답성**: 탈락(매진) 응답은 Redis 게이트에서 밀리초 단위로 반환되어야 한다.
- **일관성 모델**: 재고/결제/예약은 강한 일관성(DB 트랜잭션). Redis 잔여 수량 표시는 근사치 허용.

## 확정 사항 요약

- [x] 결제 금액은 서버가 상품 가격으로 계산하고, 클라이언트는 포인트 사용액만 지정한다.
- [x] 1 예약 요청은 수량 1개로 고정한다. 수량 N은 핵심 범위가 아니므로 제외한다.
- [x] Checkout은 남은 수량을 노출하지 않고, 예약 시도 결과로 매진 여부를 알린다.
- [x] 사용자 취소/환불 API는 범위 제외다. 단, 내부 보상 취소/환불은 구현한다.
- [x] 비기능 목표는 "Redis admission으로 매진 초과 요청의 DB write를 차단한다"와 부하 테스트 결과로 설명한다.
